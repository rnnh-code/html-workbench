All-Hands · Engineering

How we're rebuilding search.

Faster, smarter, and built per tenant.

Presenter Search Eng team
Date May 15, 2026
Audience All-hands

The problem

Search is slow, dumb, and unforgiving.

2.3s p95 latency on production search
14% of queries return zero results
0 typo tolerance — one mis-key, no answer

Current state

One cluster doing every job, badly.

Infrastructure
Single shared Elasticsearch cluster — one config for everyone.
Intelligence
No query understanding. Strings in, strings out.
Tenancy
No per-tenant tuning. Everyone gets the same relevance.

Target architecture

Hybrid retrieval, isolated per tenant.

Hybrid retrieval
Vector + lexical, fused at query time.
Per-tenant indexes
Isolated tuning, isolated blast radius.
Async reindex queue
Updates without taking the cluster down.

Rollout · Q3 2026

Four phases. Dark-launch first.

Phase 1
Dark
Shadow traffic. No user impact.
Phase 2
1%
Canary tenants. Watch dashboards.
Phase 3
10 → 50%
Staged ramp with rollback gates.
Phase 4
100%
Full cutover. Decommission legacy.

Risks & asks

Two things we need from you.

Budget approval

Per-tenant indexes plus vector embeddings. Locked in for the year if approved this quarter.
Headcount

Owns embedding model, ranking, and relevance evals. Without this, hybrid retrieval slips a quarter.
